###  Категории и ключевые риски проекта

| Категория риска                          | Конкретный риск                                                                                                                 | Вероятность (В) | Влияние (Вл) | Меры по смягчению и реагированию                                                                                                                                                |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | --------------- | ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. Технические**                       | **Низкая производительность и оптимизация**: падение FPS, лаги в VR вызывают дискомфорт и киберболезнь.                         | Высокая         | Критическое  | **Приоритизация оптимизации** с первого спринта. Использование LOD, атласов текстур. Раннее тестирование на минимальных требуемых конфигурациях PC/VR.                          |
|                                          | **Техническая сложность VR-взаимодействий**: неточный захват предметов, неестественная физика.                                  | Средняя         | Высокое      | **Создание прототипов** базовых механик на раннем этапе (движок, Leap Motion). Активное пользовательское тестирование.                                                          |
|                                          | **Проблемы совместимости**: приложение плохо работает на разных моделях VR-шлемов.                                              | Высокая         | Высокое      | Чёткое определение **целевого оборудования** в требованиях. Покупка/аренда ключевых устройств для тестов. Использование стандартизированных SDK (OpenXR).                       |
| **2. Рыночные и продуктовые**            | **Несоответствие потребностям аудитории**: интерфейс слишком сложен для новичков, не хватает нужных функций для профессионалов. | Высокая         | Критическое  | Разработка **по методологии MVP**. Ранний выпуск прототипа для закрытой бета-тестовой группы целевой аудитории. Регулярные опросы и A/B-тесты.                                  |
|                                          | **Появление сильного конкурента** или резкое изменение рыночных трендов.                                                        | Средняя         | Высокое      | **Непрерывный анализ рынка**. Фокус на своём **УТП** (например, фотореалистичная визуализация или уникальная библиотека мебели). Гибкость в адаптации плана развития продукта.  |
| **3. Организационные и операционные**    | **«Раздувание» функциональных требований (Scope Creep)**: постоянное добавление новых идей разрушает план и бюджет.             | Высокая         | Высокое      | Чёткое описание **MVP**. Введение роли **Владельца Продукта**, который приоритизирует backlog. Формальный процесс утверждения крупных изменений.                                |
|                                          | **Нехватка или выбытие ключевых специалистов** (например, разработчик под графику VR).                                          | Средняя         | Критическое  | **Перекрёстное обучение** в команде. Документирование ключевых решений. Рассмотрение аутсорса специфических задач. Формирование кадрового резерва.                              |
|                                          | **Низкая скорость интеграции**: 3D-модели от художников долго и плохо интегрируются в движок программистами.                    | Высокая         | Среднее      | Внедрение **CI/CD** pipeline для 3D-ассетов. Чёткие **технические стандарты** на модели и текстуры. Использование **Kanban-доски** для визуализации потока.                     |
| **4. Финансовые**                        | **Превышение бюджета** из-за недооценки сложности работ или роста стоимости лицензий/активов.                                   | Средняя         | Высокое      | Детализированное планирование бюджета с **резервом на риски** (10-20%). Поэтапное привлечение финансирования по результатам достижения вех (milestones).                        |
| **5. Риски для здоровья и безопасности** | **Киберболезнь (Motion Sickness) у пользователей**, ведущая к негативным отзывам и отказу от продукта.                          | Средняя         | Высокое      | Тестирование на фокус-группах с разной восприимчивостью. **Настройка комфортных параметров** по умолчанию (угол обзора, способ перемещения). Яркие предупреждения и инструкции. |
